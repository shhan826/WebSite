모달(Modality)

모달은 어떠한 작업을 완료하거나 메세지 혹은 뷰를 확인할 때까지 다른 활동을 막아 그 작업에 집중하도록 하는 역할을 합니다. acition sheet, alert, 그리고 activity view에서 모달 기능을 제공합니다. 모달 뷰가 화면에 나타나면, 사용자는 버튼을 탭해서 선택하거나 모달을 나가야만 합니다. 캘린더에서 일정을 수정하거나 Safari에서 책갈피를 선택하는 것과 같이, 일부 app에서는 모달에 기능을 넣기도 합니다. 모달 뷰는 전체 화면, 부모 뷰(parent view)의 전체부분, 혹은 팝 오버(popover)와 같이 화면 일부를 차지할 수 있습니다. 모달 뷰는 뷰를 나갈 수 있는 완료와 취소 버튼이 있는 형태가 전형적입니다.

Alert

Modal View

모달의 사용을 최소화하세요. 일반적으로 사용자는 선형적이지 않은 방법으로 app과 상호작용하는 것을 선호합니다. app 사용을 위해 작업을 완료하거나 그만두어야 하는, 혹은 중요한 정보를 저장해야 하는 경우와 같이, 주의를 꼭 끌어야만 하는 상황에서만 모달을 사용하세요.

모달 작업을 종료할 분명하고 안전한 방법을 제공하세요. 사용자가 모달 뷰를 취소했을 때의 행동의 결과가 어떻게 될지 항상 알 수 있도록 확인하세요.

모달 작업은 간단하고, 짧고, 좁은 범위를 다루어야 합니다. app 안에 새로운 app을 만들지 마세요. 모달 작업을 너무 복잡하게 만들면, 사용자는 모달을 보면서 원래 진행 중이던 작업이 무엇이었는지 잃어버리게 될 것입니다. 특히 계층적인 뷰를 포함하는 모달 작업을 만드는 경우에는 사용자가 원래 단계를 어떻게 거슬러 올라가는지 잊어버리고 길을 잃게 될 수 있으므로 신중하게 사용해야 합니다. 만약 모달 작업이 서브뷰(subview)를 포함해야 한다면, 계층적으로 하나의 길만을 만들어 완료에 이를 수 있도록 하세요. 완료 버튼은 작업을 완료할 때 사용하는 것 빼고는 사용하지 않는 것이 좋습니다.

적합한 상황이라면, 작업을 명시하는 제목을 표시하세요. 또한 뷰의 다른 부분에 텍스트를 제공해 작업에 대해 더 자세히 묘사하고 도움을 줄 수도 있습니다.

Alert(알림) 기능은 정말 필수적인(이상적으로 실행할만한) 정보 전달을 위해 남겨두세요. Alert는 app 경험을 차단하고 취소를 위해 탭을 눌러야 하므로, 사용자가 이러한 방해가 정당하다고 느껴야 할 정도로 중요합니다. 자세한 내용은 Alerts를 보세요.

알림의 기존 설정을 고려하세요. 설정에서 사용자는 app에서 알림을 어떻게 받고 싶은지 설정할 수 있습니다. 사용자가 완전히 app의 알림을 꺼버리게 하지 않도록, 기존 설정 사항을 지켜주세요.

Popover 위에 모달 뷰를 올리지 마세요. Alert에서의 가능한 예외를 제외하면, popover 위에는 어떤 것도 나타나서는 안됩니다. 정말 드물게 popover로 인해 모달 뷰가 뜨게 하는 상황이 생긴다면, 모달 뷰를 화면에 표시하기 전에 popover를 먼저 닫도록 하세요.

모달 뷰의 생김새를 app과 동등하게 하세요. 예를 들어 모달 뷰가 네비게이션 바를 포함하고 있다면, app에서 사용하는 네비게이션 바와 같은 디자인을 적용하세요.

적절한 모달 뷰 스타일을 선택하세요. 다음과 같은 스타일 중에서 선택할 수 있습니다:

풀 스크린. 화면 전체를 덮습니다. 모달 뷰 안에서 완료되어야 할 작업이 복잡할 때 사용합니다.

Page sheet. 큰 기기의 가로화면에서 콘텐츠를 일부분 가립니다. 가려지지 않은 영역은 상호작용이 안되도록 희미하게 변합니다. 큰 기기의 세로화면이나 작은 기기에서는 전체화면을 덮습니다. 모달 뷰 안에서 완료되어야 할 작업이 복잡할 때 사용합니다.

Form sheet. 화면 중앙에 나타나지만, 키보드가 있으면 위치가 변할 수 있습니다. 가려지지 않은 부분은 상호작용이 안되도록 희미하게 변합니다. 작은 기기에서는 전체 화면을 덮을 수도 있습니다. 정보를 수집할 경우에 사용합니다.

Current context. 부모 뷰(parent view)와 같은 크기로 나타납니다. 분할 뷰, popover, 혹은 전체 화면이 아닌 다른 뷰 안에서 모달을 띄울 때 사용합니다.

모달 뷰를 띄울 때 적절한 전환 효과를 선택하세요. app과 동일한 전환 효과를 사용해 일시적인 맥락 변화를 인지할 수 있도록 도우세요. 기본 효과는 모달 뷰가 아래에서 위로 미끄러져 올라오고, 취소하면 다시 아래로 내려가는 형태입니다. Flip-style 전환은 뷰를 가로 방향으로 넘겨 모달 뮤를 보여주고, 취소하면 다시 넘어가며 돌아갑니다. app 안에서는 모달 전환 효과를 일관되게 사용하세요.

모달 뷰 실행과 관련한 더 자세한 사항은 UIViewController 와 UIPresentationController를 참고하세요.
